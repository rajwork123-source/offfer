Chapter 1 — The Basics
What is Pine Script?
Pine Script is TradingView's purpose-built scripting language for creating custom indicators, strategies, and libraries directly in the browser. No installation. No compiler. You write code → click "Add to chart" → see results instantly.

❌ Pine Script cannot
Execute live orders automatically (needs bot) Connect directly to a broker API Run outside of TradingView Access live order book or trade history Replace MQL5/Python for full automation
📌
Pine Script vs MQL5 — Quick Summary
Use Pine Script if you trade on TradingView and want fast signal development with alerts. Use MQL5 if you need live automated order execution through MetaTrader with a real broker connection. Many traders use both: Pine Script for signal logic, MQL5 or Python for execution. This tutorial focuses exclusively on Pine Script v5.
Chapter 2 — Setup
TradingView Pine Editor Setup
You only need a free TradingView account to follow this tutorial. The Pine Editor is built into every chart. Here's exactly where to find everything.
1
Open any chart on TradingView
Go to tradingview.com → Charts. Open any symbol (BTCUSDT, AAPL, EURUSD — doesn't matter). The chart is your development environment. Free accounts work fine for this tutorial.
2
Find the Pine Editor tab
Look at the bottom of the chart. You'll see tabs: "Pine Editor", "Strategy Tester", and others. Click "Pine Editor". A code panel opens at the bottom of the screen.
3
Clear the default template
TradingView pre-fills a simple indicator template. Select all (Ctrl+A / Cmd+A) and delete it. You'll paste your own code here.
4
Click "Add to chart" to compile
After pasting code, click the blue "Add to chart" button (top right of Pine Editor). This compiles the script and applies it to your chart. Errors appear in a red bar at the bottom.
5
Find Strategy Tester for backtesting
When you add a strategy() script (vs indicator()), the "Strategy Tester" tab activates automatically. Click it to see trade history, equity curve, and performance metrics.

Chapter 3 — First Code
Your First Indicator
Before strategies, let's understand the basic Pine Script structure by building a simple EMA indicator. This is the "hello world" of Pine Script — get it working in 60 seconds.
my_first_indicator.pine
Pine Script v5
//@version=5 indicator("My First EMA", overlay=true) // Input: let users adjust the EMA period emaLength = input.int(20, "EMA Length", minval=1) // Calculate the EMA myEMA = ta.ema(close, emaLength) // Draw it on the chart plot(myEMA, "EMA", color=color.blue, linewidth=2)
1
//@version=5 — Always first
Every Pine Script file MUST start with this line. It tells TradingView which version of Pine Script to use. Omitting it causes TradingView to guess, which can lead to unexpected behaviour. Never skip it.
2
indicator() vs strategy()
indicator() creates a visual overlay or sub-chart — it plots things but doesn't place trades. strategy() adds buy/sell logic and activates the Strategy Tester. You cannot mix both in one script. Start with indicator() to learn, switch to strategy() when you want to backtest trades.
3
overlay=true — Which pane?
overlay=true draws on the main price chart (for things like EMAs and Bollinger Bands). overlay=false draws in a separate pane below the chart (for RSI, MACD, volume). Getting this wrong makes your chart look terrible — EMA in a sub-pane, RSI on price candles.
✅
Try It Now
Copy the code above → open TradingView → clear the Pine Editor → paste → click "Add to chart". You should see a blue EMA line on your chart. Click the gear icon on the indicator to change the length. That's Pine Script working.
Chapter 4 — Strategy
Your First Strategy
Now we switch from indicator() to strategy() — this enables the Strategy Tester and lets you see actual trade results. Here's the minimal working strategy: EMA crossover with ATR stops.
my_first_strategy.pine
Pine Script v5
//@version=5 strategy("My First Strategy", overlay=true, default_qty_type = strategy.percent_of_equity, default_qty_value = 2, commission_type = strategy.commission.percent, commission_value = 0.05) // Inputs fastLen = input.int(9, "Fast EMA") slowLen = input.int(21, "Slow EMA") // Indicators fastEMA = ta.ema(close, fastLen) slowEMA = ta.ema(close, slowLen) // Signals longEntry = ta.crossover(fastEMA, slowEMA) shortEntry = ta.crossunder(fastEMA, slowEMA) // Orders if longEntry strategy.entry("Long", strategy.long) if shortEntry strategy.entry("Short", strategy.short) // Plots plot(fastEMA, "Fast EMA", color=color.blue, linewidth=2) plot(slowEMA, "Slow EMA", color=color.orange, linewidth=2)
💡
The Key Difference: strategy.entry() vs strategy.order()
strategy.entry() — Smart. If you're already in a long and call entry("Long", strategy.long) again, it does nothing. Prevents duplicate entries automatically. Use this 95% of the time.

strategy.order() — Dumb. Fires unconditionally regardless of position. Use only when you specifically need to add to a position.
⚠️
Always Set Commission
Without the commission parameters, the backtest assumes zero trading costs — which is impossible in real life. Set commission_value=0.05 for a typical spot exchange (0.05% per trade). For futures, use 0.01–0.04%. A zero-commission backtest will always look better than live results.
Chapter 5 — Customization
Inputs & Plots
Inputs make your script adjustable without editing code. Plots visualize data on the chart. Together they make your strategy user-friendly and debuggable.
inputs_and_plots.pine
Pine Script v5
// ── Input Types ─────────────────────────────────────── intParam = input.int(14, "Integer Input", minval=1, maxval=500) floatParam = input.float(1.5, "Float Input", step=0.1) boolParam = input.bool(true, "Enable Feature") srcParam = input.source(close, "Price Source") // open/high/low/close/hl2/hlc3 tfParam = input.timeframe("240", "Higher Timeframe") colorParam = input.color(color.blue, "Line Color") // ── Plot Types ──────────────────────────────────────── // Line on main chart plot(myValue, "My Line", color=color.blue, linewidth=2) // Horizontal line (hline works outside conditions) hline(70, "Overbought", color=color.red, linestyle=hline.style_dashed) // Signal arrows on chart plotshape(longSignal, style=shape.triangleup, location=location.belowbar, color=color.green, title="Long Signal", size=size.small) // Background shading bgcolor(condition ? color.new(color.green, 90) : na) // Candle coloring barcolor(close > open ? color.green : color.red)
🎨
Transparency with color.new()
color.new(color.green, 0) = fully opaque green. color.new(color.green, 90) = 90% transparent (barely visible). The second argument is transparency 0–100. For background colors, 85–95 works well — visible but not overwhelming. For lines, use 0–20.
Chapter 6 — AI Workflow
Prompting Claude for Pine Script
The quality of your Pine Script code depends almost entirely on how you describe what you want. Here's the formula that works every time — and the mistakes that waste your time.
Perfect Prompt Template
Write a Pine Script v5 [indicator/strategy] called "[NAME]". LOGIC: - [Entry condition 1] - [Entry condition 2] - [Exit condition] SETTINGS: - Use strategy.percent_of_equity = [X]% per trade - Add stop loss: [SL logic] - Add take profit: [TP logic] - Commission: 0.05% INPUTS (make these configurable): - [Parameter 1] — default [value] - [Parameter 2] — default [value] PLOTS: - [What to draw on chart] - Background color: [condition] ALERTS: - alertcondition() for long entry, short entry IMPORTANT: overlay=[true/false], Pine Script v5 only, no lookahead bias if using request.security()
❌ Weak Prompt
"make me a pine script rsi strategy with alerts"
✅ Strong Prompt
"Write a Pine Script v5 strategy() called 'RSI Reversal'. Long when RSI(14) crosses above 30. Short when RSI crosses below 70. Stop loss 2% from entry. Take profit 4%. Position size 2% of equity. Commission 0.05%. overlay=false. Add alertcondition() for each entry."
🔑
3 Things Claude Needs for Pine Script
1. Version: Always say "Pine Script v5" — otherwise you might get deprecated v4 syntax.

2. indicator() or strategy(): Be explicit. Indicator = plots only. Strategy = trades + backtest. They can't be mixed.

3. overlay=true or false: Claude needs to know where to draw output. Price-level indicators (EMA, BB) → true. Oscillators (RSI, MACD) → false.
Chapter 7 — Debugging
Fixing Errors with AI
TradingView shows errors at the bottom of the Pine Editor when your script has a problem. Most beginners hit the same 5 errors. Here's how to diagnose and fix them instantly using Claude.
!
"Cannot call 'plot' with argument 'series bool'"
You passed a boolean (true/false) where plot() expects a number. Fix: convert it — plot(myBool ? 1 : 0) or use plotshape() for signal markers instead.
!
"Add to chart is not supported for this script type"
You're trying to use indicator() as a strategy or vice versa. Check line 2 — it should say either indicator() or strategy(), not both. Delete and repaste from scratch.
!
"Undeclared identifier"
You used a variable that was never declared. Either you typo'd the name, or it was declared inside an if-block but used outside it. Pine Script has strict scope rules — variables declared inside blocks are local to that block.
!
"Expression is evaluated incorrectly" (security error)
Lookahead bias in request.security(). Always add barmerge.lookahead_off as the last argument. This is the #1 cause of backtests that look amazing but fail live.
Error Fix Prompt Template
I have a Pine Script v5 error. Error message: "[PASTE EXACT ERROR TEXT HERE]" Error is on or near line: [LINE NUMBER] Full script: [PASTE YOUR ENTIRE CODE HERE] Please: 1. Fix the error 2. Explain what caused it in plain English 3. Show me how to avoid this type of error in future
Chapter 8 — Backtesting
Using the Strategy Tester
Once your strategy() script is on the chart, the Strategy Tester tab activates. This is TradingView's built-in backtesting engine — it shows every historical trade and calculates performance metrics automatically.
1
Click the "Strategy Tester" tab
It appears next to the Pine Editor tab at the bottom of the screen — but only when your script uses strategy(), not indicator(). Click it to open the backtest panel.
2
Use "Performance Summary" for key metrics
The three tabs: Overview (quick summary), Performance Summary (all metrics in detail), and List of Trades (every trade in sequence). Start with Performance Summary.
3
Set the date range for your test period
Click the calendar icon in Strategy Tester to set a specific date range. Always test on at least 3 years of data. Use "Properties" to set initial capital and commission if you didn't hardcode them.
4
Check the Equity Curve
The equity curve plots your account value over time. A healthy curve trends upward with manageable dips. A curve that crashes 50% and then recovers — that's a strategy you can't emotionally trade live, regardless of the final number.
🚨
The Lookahead Bias Warning
If your strategy uses request.security() and you didn't set barmerge.lookahead_off, the backtest may look incredible because it's seeing the future. This is the single most common reason a Pine Script backtest fails to replicate in live trading. Always explicitly pass barmerge.lookahead_off — never rely on the default.
Chapter 9 — Metrics
Reading Backtest Results
The Strategy Tester shows dozens of numbers. Most are noise. Here are the 6 that actually matter — and what to look for in each.
>1.5
Profit Factor target
>150
Min total trades
<25%
Max drawdown
+ve
Average trade P&L
📊
Profit Factor — the most important number
Gross profit ÷ gross loss. Above 1.0 = profitable. Above 1.5 = solid. Above 2.0 = very good (but check for overfitting). Under 1.2 = too marginal for live trading after slippage. Optimize for this metric, not net profit.
📊
Max Drawdown — the emotional gut-check
The biggest peak-to-trough drop in equity during the test. Live trading always has higher drawdown than backtests. If the backtest shows 20% drawdown, expect 25–30% live. If you can't emotionally handle that drawdown, this strategy isn't for you regardless of total return.
📊
Win Rate — context matters
Win rate alone is meaningless. A 30% win rate strategy with 10:1 risk-reward beats a 70% win rate strategy with 1:2 risk-reward. Always look at win rate AND average win/loss size together. A small average win with a large average loss = eventual blowup even with high win rate.
📊
Total Trades — statistical significance
Under 50 trades = results are mostly luck. You need at least 150–200 trades for a backtest to have statistical meaning. If your strategy only generated 30 trades over 3 years, either test on a shorter timeframe or accept the results are unreliable.
⚠️
Backtest ≠ Live Performance
Backtests always look better than live because: (1) you're using perfect historical data with no slippage beyond what you set, (2) you know the strategy survived — survivorship bias, (3) bars close at the exact price you need. In live trading, spreads, requotes, and data gaps eat into results. A backtest Profit Factor of 1.8 might become 1.3–1.5 live. Plan for degradation.
Chapter 10 — Ship It
First Strategy Checklist
Before you enable real-money alerts from your Pine Script strategy, tick every item below. Click each one as you complete it.
Code Quality
Script loads with zero errors in Pine Editor. No red messages in the status bar at all.
First line is //@version=5. Not v4, not missing, exactly this.
All key values are input() parameters, not hardcoded. EMA lengths, stop loss %, take profit %, etc.
Commission is set in strategy() — commission_type and commission_value are not left at zero.
Backtest Quality
Tested on at least 2–3 years of data across different market conditions (trending and ranging periods).
Profit Factor above 1.4 consistently, not just in the best date range.
More than 150 total trades in the backtest — otherwise results aren't statistically meaningful.
Max drawdown under 25%. If higher, reduce position size or tighten stop losses.
Tested on out-of-sample data — optimized on one date range, validated on a different one.
Alerts & Risk
alertcondition() calls added for all entry signals. Tested by watching the chart live.
Stop loss defined for every trade via strategy.exit(). Never running naked positions.
Paper traded for at least 2 weeks with live alerts but no real money. Signals match expectations.
