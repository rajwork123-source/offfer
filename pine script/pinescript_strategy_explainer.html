EMA Crossover Strategy
The EMA Crossover is the "hello world" of algorithmic trading. When a fast-moving average crosses above a slow one, it signals the start of an uptrend. When it crosses below, a downtrend. In Pine Script, this is elegantly expressed in a single line using ta.crossover() — but understanding why each surrounding line exists is what separates a vibe coder from a vibe guesser.
Long Entry
9 EMA crosses above 21 EMA
Short Entry
9 EMA crosses below 21 EMA
Exit
Opposite signal (reverse)
Best Market
Trending markets, daily/4H
1
Version declaration & strategy() header
//@version=5 strategy("EMA Cross", overlay=true, default_qty_type=strategy.percent_of_equity, default_qty_value=2)
//@version=5
Required first line. Tells TradingView which Pine Script version to use. Always use version 5 — it has better syntax, more features, and ongoing support. Omitting this causes TradingView to guess the version, which can lead to silent bugs. v4 and v5 have breaking differences: v4 uses study(), v5 uses indicator(). Always be explicit.
overlay=true
Draws on the main price chart, not a separate pane. Since we're plotting EMAs directly on price, they need to live on the same chart. If you were plotting RSI (which has its own 0–100 scale), you'd use overlay=false. Getting this wrong is the most common first-time Pine Script mistake. Wrong overlay = unusable chart layout.
default_qty_type=strategy.percent_of_equity
Sets position sizing mode to a percentage of account equity. This means every trade risks the same percentage of your current balance, not a fixed dollar amount. As your account grows, position size grows proportionally. Alternatives: strategy.fixed (fixed contracts) or strategy.cash (fixed dollar amount). Percent of equity is the professional choice — it compounds correctly and automatically adjusts as your balance changes.
default_qty_value=2
The percentage — 2% of equity per trade. Combined with the type above, each trade risks 2% of the current account balance. On a $10,000 account, that's $200 per trade. Conservative traders use 1%; aggressive use up to 5%. This is the most important risk parameter in your entire strategy. Never start with more than 2% until you have at least 6 months of live results.
2
Inputs & Indicators
// Inputs — configurable parameters fastLen = input.int(9, "Fast EMA Length", minval=1) slowLen = input.int(21, "Slow EMA Length", minval=1) // Compute the two EMAs fastEMA = ta.ema(close, fastLen) slowEMA = ta.ema(close, slowLen)
input.int(9, "Fast EMA Length", minval=1)
Creates a user-adjustable input field in the strategy settings panel. The 9 is the default value. The label "Fast EMA Length" appears in the settings UI. The minval=1 prevents someone entering 0 or negative, which would cause an error. Why use inputs instead of hardcoding 9? Because inputs let you use TradingView's Optimizer to test multiple values automatically without editing code every time.
ta.ema(close, fastLen)
Computes an Exponential Moving Average of the closing price. ta. is Pine Script's technical analysis library — it contains EMA, SMA, RSI, MACD, Bollinger Bands, and dozens more. The first argument is the source series (usually close) and the second is the lookback period. In Pine Script, ta.ema() automatically runs on every bar in the chart's history — you don't need a loop. This is Pine Script's "series" model and is the biggest conceptual difference from other languages.
3
Signal Detection & Entries
// Detect crossovers longSignal = ta.crossover(fastEMA, slowEMA) shortSignal = ta.crossunder(fastEMA, slowEMA) // Place orders on signals if longSignal strategy.entry("Long", strategy.long) if shortSignal strategy.entry("Short", strategy.short)
ta.crossover(fastEMA, slowEMA)
Returns true only on the one bar where fastEMA crosses FROM BELOW to ABOVE slowEMA. It compares the current bar with the previous bar — if fast was below slow before and is now above, it returns true. On every other bar, it returns false. This is the core signal. The mirror function ta.crossunder() is the reverse — true only when fast crosses from above to below slow.
strategy.entry("Long", strategy.long)
Places a market buy order and records it in the Strategy Tester. The first argument is the order ID string — used to match entries with exits. The second is the direction: strategy.long or strategy.short. If you're already in a long position when this fires, Pine Script closes it first by default. You don't need to manually check if a position is open — strategy.entry() handles that. But strategy.order() does not — subtle difference that trips up beginners.
4
Plots & Alerts
// Draw EMAs on chart plot(fastEMA, "Fast EMA", color=color.new(color.blue, 0), linewidth=2) plot(slowEMA, "Slow EMA", color=color.new(color.orange, 0), linewidth=2) // Background color: green in long, red in short bgcolor(strategy.position_size > 0 ? color.new(color.green, 90) : strategy.position_size < 0 ? color.new(color.red, 90) : na) // Alert conditions alertcondition(longSignal, "Long Entry", "EMA Cross LONG — {{ticker}} @ {{close}}") alertcondition(shortSignal, "Short Entry", "EMA Cross SHORT — {{ticker}} @ {{close}}")
plot(fastEMA, "Fast EMA", color=color.new(color.blue, 0))
Draws the EMA line on the chart. color.new(color.blue, 0) creates an opaque blue (0% transparency). To make it semi-transparent, use a value like 60. The "Fast EMA" string is the legend label in the chart. plot() only works at the script level, not inside if-blocks. If you need conditional plots, use plotshape() or set the color to na (invisible) when inactive.
strategy.position_size > 0
A built-in variable that tells you the current net position size. Positive = long position open. Negative = short position. Zero = flat (no position). This powers the background color — it turns green during long trades and red during shorts, making it instantly obvious when the strategy was in the market. Using position_size instead of tracking signals manually is simpler and handles edge cases like partial exits automatically.
alertcondition(longSignal, "Long Entry", "...")
Creates a named alert template that users can subscribe to in TradingView's Alert dialog. When longSignal is true on a bar, TradingView fires the alert. The third argument is the message, which supports dynamic variables like {{ticker}} and {{close}}. alertcondition() creates the template — you still need to go to TradingView → Alerts → Create and select it. The alternative alert() function fires immediately from code without requiring setup.
💡
Why EMA and not SMA? A Simple Moving Average gives equal weight to every bar in the period. An EMA gives exponentially more weight to recent bars. This means the EMA reacts faster to new price movement — crossover signals appear earlier. Faster reaction is a double-edged sword: you enter trends sooner, but you also get more false signals in choppy, sideways markets.
⚠️
The biggest weakness of EMA crossover strategies is "whipsawing" — in choppy markets, price oscillates around the crossover level repeatedly, generating many false signals. Always add a trend filter (like requiring ADX > 25) or use a higher timeframe EMA as a directional filter before trading this with real money.
ema_cross_strategy.pine — complete v5 code
//@version=5 strategy("EMA Cross Strategy", overlay=true, default_qty_type=strategy.percent_of_equity, default_qty_value=2, commission_type=strategy.commission.percent, commission_value=0.05) // ── Inputs ─────────────────────────────────────────────── fastLen = input.int(9, "Fast EMA Length", minval=1) slowLen = input.int(21, "Slow EMA Length", minval=1) atrLen = input.int(14, "ATR Length") slMult = input.float(1.5, "SL ATR Multiplier") tpMult = input.float(3.0, "TP ATR Multiplier") // ── Indicators ─────────────────────────────────────────── fastEMA = ta.ema(close, fastLen) slowEMA = ta.ema(close, slowLen) atr = ta.atr(atrLen) // ── Signals ────────────────────────────────────────────── longSignal = ta.crossover(fastEMA, slowEMA) shortSignal = ta.crossunder(fastEMA, slowEMA) // ── Entries with ATR-based SL/TP ──────────────────────── if longSignal strategy.entry("Long", strategy.long) strategy.exit("Long Exit", "Long", stop = close - atr * slMult, limit = close + atr * tpMult) if shortSignal strategy.entry("Short", strategy.short) strategy.exit("Short Exit", "Short", stop = close + atr * slMult, limit = close - atr * tpMult) // ── Plots ──────────────────────────────────────────────── plot(fastEMA, "Fast EMA", color=color.blue, linewidth=2) plot(slowEMA, "Slow EMA", color=color.orange, linewidth=2) bgcolor(strategy.position_size > 0 ? color.new(color.green, 90) : strategy.position_size < 0 ? color.new(color.red, 90) : na) // ── Alerts ─────────────────────────────────────────────── alertcondition(longSignal, "Long Entry", "LONG: {{ticker}} @ {{close}}") alertcondition(shortSignal, "Short Entry", "SHORT: {{ticker}} @ {{close}}")
Oscillator Mean Reversion
RSI Overbought / Oversold Strategy
The RSI (Relative Strength Index) measures the speed and magnitude of recent price changes, producing a value between 0 and 100. Above 70 = overbought (rose too fast, pullback likely). Below 30 = oversold (fell too fast, bounce likely). This Pine Script strategy bets on those reversals — entering when the RSI crosses back through the threshold, not merely touches it.
Buy Signal
RSI crosses above 30 (from oversold)
Sell Signal
RSI crosses below 70 (overbought)
RSI Period
14 bars (standard)
Best Market
Ranging / sideways conditions
1
RSI Computation
rsiLen = input.int(14, "RSI Length", minval=2) oversold = input.int(30, "Oversold Level", minval=1, maxval=49) overbought = input.int(70, "Overbought Level", minval=51, maxval=99) rsi = ta.rsi(close, rsiLen)
ta.rsi(close, rsiLen)
Computes the RSI in one line. Pine Script's ta.rsi() handles all the smoothed average gain/loss calculations internally. You just supply the source series (close) and the lookback period (14). The result is a series of values between 0 and 100, recalculated on every bar. In Pine Script v4 you'd use rsi() — in v5, all technical indicators live in the ta. namespace. Always use ta.rsi() to avoid deprecation warnings.
minval=1, maxval=49
Guards on inputs prevent nonsensical values. An oversold level must be below 50 (the midline). An overbought level must be above 50. Without these constraints, a user could accidentally set both thresholds the same, causing the strategy to never trade or always trade. Always add minval and maxval to numeric inputs — it makes your script bulletproof and shows up in the Settings panel as a slider range.
2
Signal Logic — Crossover vs Simple Comparison
// WRONG — fires every bar RSI is below 30 // wrongLong = rsi < oversold // CORRECT — fires only on the bar RSI crosses BACK above 30 longSignal = ta.crossover(rsi, oversold) shortSignal = ta.crossunder(rsi, overbought)
ta.crossover(rsi, oversold)
This fires true only on the bar where RSI crosses FROM BELOW 30 to ABOVE 30. If RSI has been below 30 for 5 bars and you used rsi < 30, the strategy would try to enter 5 times. With ta.crossover(), it enters once — on the recovery, not the initial dip. This is the single most important distinction in Pine Script signal design. rsi < 30 is a "state" (true for many bars). ta.crossover() is an "event" (true for exactly one bar). Strategies should almost always be triggered by events, not states.
⚠️
RSI mean reversion fails catastrophically in trending markets. When a market is in a strong uptrend, RSI can stay above 70 for weeks — shorting every time it crosses below 70 will produce a string of losses. Always add a trend filter: only take RSI long signals when price is above its 200-period SMA, and only shorts when below.
rsi_mean_reversion.pine — complete v5 code
//@version=5 strategy("RSI Mean Reversion", overlay=true, default_qty_type=strategy.percent_of_equity, default_qty_value=2, commission_type=strategy.commission.percent, commission_value=0.05) rsiLen = input.int(14, "RSI Length") oversold = input.int(30, "Oversold Level", minval=1, maxval=49) overbought = input.int(70, "Overbought Level", minval=51, maxval=99) slPct = input.float(2.0, "Stop Loss %", minval=0.1) tpPct = input.float(4.0, "Take Profit %", minval=0.1) rsi = ta.rsi(close, rsiLen) longSignal = ta.crossover(rsi, oversold) shortSignal = ta.crossunder(rsi, overbought) if longSignal strategy.entry("Long", strategy.long) strategy.exit("Long Exit", "Long", stop = close * (1 - slPct / 100), limit = close * (1 + tpPct / 100)) if shortSignal strategy.entry("Short", strategy.short) strategy.exit("Short Exit", "Short", stop = close * (1 + slPct / 100), limit = close * (1 - tpPct / 100)) // RSI pane plot rsiPlot = plot(rsi, "RSI", color=color.blue) obLine = hline(overbought, "Overbought", color=color.red, linestyle=hline.style_dashed) osLine = hline(oversold, "Oversold", color=color.green, linestyle=hline.style_dashed) fill(rsiPlot, obLine, color=color.new(color.red, 90)) fill(rsiPlot, osLine, color=color.new(color.green, 90)) alertcondition(longSignal, "RSI Long", "RSI LONG: {{ticker}} — RSI crossed above 30") alertcondition(shortSignal, "RSI Short", "RSI SHORT: {{ticker}} — RSI crossed below 70")
Volatility Breakout
Bollinger Band Squeeze Breakout
Bollinger Bands envelope price with a standard deviation channel. When the bands narrow (the "squeeze"), volatility is contracting — energy is building. When price breaks out of the narrow bands, it often moves explosively. This strategy detects squeezes and trades the breakout direction, using ATR to size stops.
Squeeze Detect
BB width below 20-bar minimum
Long Entry
Price closes above upper BB during squeeze
Short Entry
Price closes below lower BB during squeeze
Exit
ATR-based stop and 2:1 take profit
1
Bollinger Band Computation
bbLen = input.int(20, "BB Length") bbMult = input.float(2.0, "BB Multiplier") sqzBars = input.int(20, "Squeeze Lookback") [bbMiddle, bbUpper, bbLower] = ta.bb(close, bbLen, bbMult) bbWidth = bbUpper - bbLower squeeze = bbWidth == ta.lowest(bbWidth, sqzBars)
[bbMiddle, bbUpper, bbLower] = ta.bb(...)
ta.bb() returns three values simultaneously using tuple destructuring. This Pine Script v5 syntax assigns the middle band (SMA), upper band (SMA + N×StdDev), and lower band (SMA − N×StdDev) to three variables in one line. In v4, you'd call ta.bb() separately for each value. The tuple syntax in v5 is cleaner and more efficient — it only computes the Bollinger Bands once.
bbWidth == ta.lowest(bbWidth, sqzBars)
This detects the squeeze: true when the current band width equals the lowest width seen in the last 20 bars. ta.lowest() returns the minimum value of a series over the lookback period. When band width hits a new 20-bar low, the bands are at their narrowest — this is the squeeze condition. The squeeze fires on the exact bar where width is the tightest it's been. The breakout logic then checks if price has moved outside the bands on that same bar — energy released.
bollinger_squeeze.pine — complete v5 code
//@version=5 strategy("Bollinger Squeeze Breakout", overlay=true, default_qty_type=strategy.percent_of_equity, default_qty_value=2, commission_type=strategy.commission.percent, commission_value=0.05) bbLen = input.int(20, "BB Length") bbMult = input.float(2.0, "BB Multiplier") sqzBars = input.int(20, "Squeeze Lookback") atrLen = input.int(14, "ATR Length") slMult = input.float(1.5, "SL ATR Mult") [bbMid, bbUp, bbLow] = ta.bb(close, bbLen, bbMult) bbWidth = bbUp - bbLow atr = ta.atr(atrLen) squeeze = bbWidth == ta.lowest(bbWidth, sqzBars) longSignal = squeeze and close > bbUp shortSignal = squeeze and close < bbLow if longSignal strategy.entry("Long", strategy.long) strategy.exit("Long Exit", "Long", stop=close - atr * slMult, limit=close + atr * slMult * 2) if shortSignal strategy.entry("Short", strategy.short) strategy.exit("Short Exit", "Short", stop=close + atr * slMult, limit=close - atr * slMult * 2) plot(bbUp, "Upper BB", color=color.blue) plot(bbMid, "Middle BB", color=color.gray) plot(bbLow, "Lower BB", color=color.blue) bgcolor(squeeze ? color.new(color.yellow, 90) : na) alertcondition(longSignal, "BB Squeeze Long", "BB Squeeze BREAKOUT LONG: {{ticker}}") alertcondition(shortSignal, "BB Squeeze Short", "BB Squeeze BREAKOUT SHORT: {{ticker}}")
Momentum Histogram
MACD Histogram Divergence
MACD (Moving Average Convergence Divergence) measures the relationship between two EMAs. The histogram — the difference between MACD and its signal line — is the most sensitive part. When the histogram flips from negative to positive, momentum is shifting bullish. This strategy trades those histogram zero-line crossings with a trend filter.
Long Entry
MACD histogram crosses above zero
Short Entry
MACD histogram crosses below zero
Settings
12, 26, 9 (standard)
Filter
200 EMA trend direction
macd_histogram.pine — complete v5 code
//@version=5 strategy("MACD Histogram", overlay=true, default_qty_type=strategy.percent_of_equity, default_qty_value=2, commission_type=strategy.commission.percent, commission_value=0.05) fastMacd = input.int(12, "MACD Fast") slowMacd = input.int(26, "MACD Slow") signalLen = input.int(9, "Signal Length") trendLen = input.int(200, "Trend EMA Length") [macdLine, signalLine, hist] = ta.macd(close, fastMacd, slowMacd, signalLen) trendEMA = ta.ema(close, trendLen) // Histogram zero-line crossings with trend filter longSignal = ta.crossover(hist, 0) and close > trendEMA shortSignal = ta.crossunder(hist, 0) and close < trendEMA if longSignal strategy.entry("Long", strategy.long) if shortSignal strategy.entry("Short", strategy.short) // Close on opposite signal if shortSignal and strategy.position_size > 0 strategy.close("Long") if longSignal and strategy.position_size < 0 strategy.close("Short") plot(trendEMA, "200 EMA", color=color.white, linewidth=2) plotshape(longSignal, style=shape.triangleup, location=location.belowbar, color=color.green, size=size.small) plotshape(shortSignal, style=shape.triangledown, location=location.abovebar, color=color.red, size=size.small) alertcondition(longSignal, "MACD Long", "MACD Histogram LONG: {{ticker}} @ {{close}}") alertcondition(shortSignal, "MACD Short", "MACD Histogram SHORT: {{ticker}} @ {{close}}")
Multi-Timeframe Advanced
Multi-Timeframe Trend Filter
The professional-grade approach: use a higher timeframe for trend direction and a lower timeframe for entry precision. This script reads the 4-hour EMA to determine the macro trend, then takes 15-minute EMA crossover entries that align with it — filtering out counter-trend trades that lose more often than not.
Trend TF
4H — 200 EMA direction
Entry TF
15m — 9/21 EMA crossover
Long Filter
Price above 4H 200 EMA only
Key Concern
Lookahead bias prevention
1
request.security() — fetching higher timeframe data
// WRONG — causes lookahead bias (future leak) // htfEMA = request.security(syminfo.tickerid, "240", ta.ema(close, 200)) // CORRECT — no lookahead htfEMA = request.security(syminfo.tickerid, "240", ta.ema(close, 200), barmerge.gaps_off, barmerge.lookahead_off)
request.security(syminfo.tickerid, "240", ...)
Fetches a series from a different timeframe for the same symbol. The timeframe is specified as minutes: "240" = 4 hours, "60" = 1 hour, "D" = daily. syminfo.tickerid automatically uses the current chart's symbol, so the strategy works on any symbol. request.security() is Pine Script's most powerful and most misused feature. It lets you combine multiple timeframes in one script — but used incorrectly, it looks at future candles and produces backtest results that can't occur in live trading.
barmerge.lookahead_off
Critical parameter that prevents lookahead bias. With lookahead_off, when running on a 15m chart, the 4H EMA value used is the value at the most recently COMPLETED 4H bar — not the current, in-progress one. This matches what would be available in real-time trading. Without barmerge.lookahead_off (the default changed in v5 to be off, but always be explicit), the strategy can peek at partial future 4H candles. This is the #1 cause of "too-good-to-be-true" multi-timeframe backtests that fail live.
mtf_trend_filter.pine — complete v5 code
//@version=5 strategy("MTF Trend Filter Strategy", overlay=true, default_qty_type=strategy.percent_of_equity, default_qty_value=2, commission_type=strategy.commission.percent, commission_value=0.05) // Inputs fastLen = input.int(9, "Fast EMA") slowLen = input.int(21, "Slow EMA") htfTF = input.timeframe("240", "Higher TF (minutes)") htfLen = input.int(200, "HTF EMA Length") atrLen = input.int(14, "ATR Length") slMult = input.float(1.5, "SL ATR Mult") // Local timeframe EMAs fastEMA = ta.ema(close, fastLen) slowEMA = ta.ema(close, slowLen) atr = ta.atr(atrLen) // Higher timeframe EMA — NO lookahead htfEMA = request.security(syminfo.tickerid, htfTF, ta.ema(close, htfLen), barmerge.gaps_off, barmerge.lookahead_off) // Trend filter bullTrend = close > htfEMA bearTrend = close < htfEMA // Entry signals — only with trend longSignal = ta.crossover(fastEMA, slowEMA) and bullTrend shortSignal = ta.crossunder(fastEMA, slowEMA) and bearTrend if longSignal strategy.entry("Long", strategy.long) strategy.exit("Long Exit", "Long", stop=close - atr * slMult, limit=close + atr * slMult * 2) if shortSignal strategy.entry("Short", strategy.short) strategy.exit("Short Exit", "Short", stop=close + atr * slMult, limit=close - atr * slMult * 2) // Plots plot(fastEMA, "Fast EMA", color=color.blue, linewidth=1) plot(slowEMA, "Slow EMA", color=color.orange, linewidth=1) plot(htfEMA, "HTF EMA", color=color.white, linewidth=3) bgcolor(bullTrend ? color.new(color.green, 96) : bearTrend ? color.new(color.red, 96) : na) alertcondition(longSignal, "MTF Long", "MTF LONG: {{ticker}} — trend + EMA cross") alertcondition(shortSignal, "MTF Short", "MTF SHORT: {{ticker}} — trend + EMA cross")
