Step 1 of 6
Create Your TradingView Account
TradingView is your IDE, your charting platform, and your script host all in one. No download needed — it runs at tradingview.com.
A
Go to TradingView
Open your browser and navigate
1
Open any modern browser (Chrome, Firefox, Edge, Safari). Go to tradingview.com.
2
Click the Get Started button in the top right corner of the landing page.
3
You'll be shown three options: sign up with Google, Apple, or email. Pick whichever you prefer — all three work the same.
ℹ️
Tip: Use a Google account if you have one. It speeds up login and you'll never forget a password. TradingView supports Google OAuth natively.
B
Complete Registration
Set username and verify email
1
Choose a username. This is your public identity if you publish scripts. Pick something professional — it cannot be easily changed later.
2
If you signed up with email, check your inbox for a verification email from TradingView. Click the link inside it.
3
Skip the plan selection for now — select Basic (free). You can upgrade later. All Pine Script features are available on Basic for personal use.
4
You'll land on the main chart page. This is your home base — everything starts here.
C
Set Up Your Chart Layout
Configure the canvas you'll code against
1
In the top search bar, type SPY or BTCUSDT (whichever market you care about) and press Enter. This loads the chart.
2
Switch timeframe: click the timeframe selector at the top (e.g., 1D) and choose 1H (1 hour). Good for testing indicators — enough bars, not too noisy.
3
Right-click the chart → Chart Properties → uncheck Ask before removing studies. Saves a click every time you iterate.
4
Press Ctrl+S (or Cmd+S on Mac) to save your chart layout. Name it something like "Dev Chart".
💡
Pro tip: Create a dedicated "Dev" chart layout separate from your real trading charts. This way you can experiment freely without cluttering your analysis workspace.

Step 3 of 6
Open the Pine Script Editor
The Pine Editor is TradingView's built-in IDE. It lives at the bottom of your chart. Here you'll write, save, compile, and add scripts to your chart — all in one place.
A
Open the Editor
Two ways to open it
1
Method 1 (Recommended): At the very bottom of the TradingView chart page, find the footer toolbar. Click Pine Editor tab. The editor panel slides up.
2
Method 2: Press Alt+P (Windows) or Option+P (Mac) from anywhere on the chart page to toggle the Pine Editor.
3
Drag the top edge of the editor panel upward to give it more vertical space. Aim for roughly half the screen height — you want to see both chart and code simultaneously.
Pine Editor — TradingView
// Pine Script v6 — default template
//@version=6
indicator("My Script", overlay=true)
plot(close)
B
Editor Interface Walkthrough
Every button and what it does
1
Top-left: Script name dropdown — click to rename your script, create a new one, or switch between saved scripts.
2
Add to chart button — compiles your code and plots it on the chart. If there are errors, a red error panel appears at the bottom of the editor instead.
3
Save button (💾 or Ctrl+S) — saves your script to TradingView's cloud. On the free plan, this overwrites your one saved slot.
4
Publish Script — makes your script visible on the public library. Skip this for now; publishing requires a chart screenshot and description.
5
Error/Warning panel — appears below the code area when compilation fails. Clicking an error jumps the cursor to the problematic line.
💡
Keyboard shortcut: Ctrl+Enter (or Cmd+Enter) is the fastest way to compile and add your script to the chart. Use this constantly as you develop — it's faster than clicking.
C
Essential Keyboard Shortcuts
Memorize these first
→
Ctrl+Enter — Add to chart (compile + run)
→

→

→

→
Ctrl+Space — Trigger autocomplete manually
→

Step 4 of 6
Write Your First Line of Pine Script
Before learning the language in depth, let's verify your environment works end-to-end. You'll write a minimal indicator and confirm it runs on the chart.
A
Clear the Editor and Type This
Your first working indicator
1
Select all text in the editor (Ctrl+A) and delete it. Start from a blank slate.
2
Type the following code by hand — don't copy-paste yet. Typing it helps you learn the structure.
Pine Script v6
// My first Pine Script indicator //@version=6 indicator("Hello Pine Script", overlay=true) // Plot the closing price on the chart plot(close, title="Close Price", color=color.blue, linewidth=2) // Plot a 20-period Simple Moving Average sma20 = ta.sma(close, 20) plot(sma20, title="SMA 20", color=color.orange, linewidth=2)
3
Press Ctrl+Enter. The script compiles and two lines appear on your chart: the close price in blue and the SMA in orange.
✅
If you see two lines on the chart, your Pine Script environment is working perfectly. If you see red error text in the editor, re-read the code carefully — a single typo can cause compilation failure.
B
Understand What You Just Wrote
Line-by-line breakdown
1
//@version=6 — mandatory declaration. Tells TradingView which version of Pine Script to compile with. Always use version 6 (the latest).
2
indicator(...) — declares this script as an indicator (not a strategy or library). overlay=true means plot on top of the price chart, not in a separate pane below.
3
close — a built-in variable. Contains the closing price of each bar. Pine Script has many like this: open, high, low, volume, time.
4
ta.sma(close, 20) — calls the technical analysis built-in sma() function. First arg is the data series, second is the period. Returns a series of SMA values, one per bar.
5
plot() — renders a value on the chart as a line. It runs once per bar, left to right, drawing the full series.
Step 5 of 6
Add to Chart, Save & Publish
Now that your script runs, learn the full workflow: adding it to different charts, saving it to the cloud, and optionally publishing it to the TradingView public library.
A
The Add to Chart Workflow
What happens every time you press Ctrl+Enter
1
TradingView compiles your Pine code. If there are syntax errors, it stops here and shows error messages. Fix them and try again.
2
The compiled script is executed against the chart's data. Pine runs your script bar-by-bar, from the oldest bar to the most recent.
3
Output (plots, shapes, labels, lines, alerts) is rendered on the chart. The chart automatically rescales to fit new plotted values.
4
The script name appears in the indicators list at the top of the chart. You can click its name to open settings, or click the × to remove it.
💡
Indicator vs Strategy: If you use indicator(), the script only plots. If you use strategy(), it also simulates trades and populates the Strategy Tester tab at the bottom. You don't need the Strategy Tester until you're writing strategies.
B
Saving Your Script
Cloud save vs. local backup
1
Press Ctrl+S or click the Save button. Your script is uploaded to TradingView's cloud and accessible from any browser.
2
On the free plan, you have 1 saved script slot. Saving a new script overwrites the previous one. Workaround: maintain a local .pine text file as backup.
3
To keep multiple versions locally: open Notepad (or VS Code), paste your Pine code, and save as my_indicator_v1.pine. Use VS Code with the Pine Script Syntax Highlighter extension for a better local experience.
C
Publishing to the Public Library
Optional — share with the community
1
Click Publish Script in the editor. Choose Open Script (full source visible) or Protected Script (compiled only, requires paid plan).
2
Add a chart screenshot showing your indicator in action. TradingView uses this as the thumbnail in the public library — make it clear and descriptive.
3
Write a description explaining what your script does, the signals it generates, and how to use it. Good descriptions get more followers and boosted visibility.
💡
Don't rush to publish: Published scripts go through a review moderation queue. TradingView moderators check that the script works, isn't a copy, and has proper documentation. Focus on learning first — publish when you have something genuinely useful to share.
Step 6 of 6
Settings & Best Practices
Small configuration choices now save hours of frustration later. Lock these in before you start building real scripts.
A
TradingView Chart Settings
Optimize the canvas for development
1
Right-click chart → Chart Settings → Appearance: Set background to dark. Dark mode reduces eye strain during long coding sessions and makes plotted lines easier to see.
2
Go to Chart Settings → Scales: Enable Indicators and financials scale labels. This shows the current value of each indicator next to the price scale — makes debugging plots much easier.
3
Right-side scale: right-click the price scale → check Scale price chart only. This prevents non-overlay indicators (like RSI) from distorting the price scale.
B
Pine Script Version Convention
Always use v6
1
Always start scripts with //@version=6. This is Pine Script version 6, the current stable release with the most features and best performance.
2
When you read community scripts, many use //@version=5 or older. Most v5 code runs in v6 with minor changes. Use the migration guide in TradingView docs if you need to port old code.
3
Never mix version annotations — don't write //@version=5 and use v6-only features. The compiler will tell you when there's a mismatch.
C
Development Best Practices
Habits that make you a better Pine coder
1
Test on multiple timeframes. A script that looks great on 1H might be useless on 15m. Always verify your logic holds across different resolutions before shipping.
2
Test on multiple symbols. Write your script against BTCUSDT, verify it also works on EURUSD and SPY. Symbol-specific quirks (gaps, halts, low volume) expose bugs fast.
3
Comment your code. Pine scripts can get complex. A comment explaining why you chose a particular length or threshold saves future-you hours of re-reverse-engineering your own logic.
4
Use input() for all constants. Never hardcode numbers like 14 or 200 in your logic. Use input.int(14, "RSI Length") so users (and you) can adjust them from the Settings panel without touching code.
5
Keep scripts in a local folder. Save every version of every script as scriptname_vX.pine. Free plan users especially — cloud storage is limited, local backup is unlimited.
