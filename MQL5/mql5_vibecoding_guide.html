Chapter 02 — Prompt Engineering
How to talk to AI about
trading code
The difference between getting working MQL5 code and getting garbage is 95% about how you frame your prompt. Here's the exact framework that works.
The Anatomy of a Great Prompt
Always include these 5 things
1
Context: What platform and language
Always start with "Write MQL5 code for MetaTrader 5". Without this, AI might write Python, Pine Script (TradingView), or C++ that won't compile in MetaEditor. Be explicit every single time.
2
Strategy logic: Exactly when to buy and sell
Don't say "make a moving average strategy." Say "buy when the 9-period EMA crosses above the 21-period EMA on the H1 chart, close the position when it crosses back below." Precision = better code.
3
Risk management: Stop loss, take profit, lot size
Always specify your risk rules. "Use a 50-pip stop loss and 100-pip take profit with 0.1 lot size as an input parameter." If you don't specify, AI might generate an EA with no stop loss at all — which is dangerous.
4
Filters: Conditions that must be true before trading
Good strategies don't trade all the time. "Only trade during London and New York session overlap (13:00–17:00 UTC)" or "only trade if ADX is above 25 confirming a trend."
5
Quality requirements: Best practices
End every prompt with: "Use CTrade class, check indicator handle validity, use input parameters for all key values, include OnDeinit to release handles, follow MQL5 best practices."
Real Prompt Examples
Bad vs Good prompts
✗ Bad Prompt — Vague
Make me a Bollinger Bands EA for MT5 that makes money.
✓ Good Prompt — Precise
Write MQL5 Expert Advisor code for MetaTrader 5 that: - Uses Bollinger Bands (20,2.0) - Buys when previous closed bar closes below the lower band and current bar opens above it - Exits when price reaches the middle band (SMA 20) - Stop loss: 1.5x band width - Lot size: input parameter - No more than 1 open trade - Use CTrade, release handles in OnDeinit

Chapter 04 — Test It
Backtesting with
Strategy Tester
Backtesting runs your EA against historical price data as if it traded live. This tells you how it would have performed — before risking a single real dollar.
Setting Up the Backtest
The right backtesting setup
In MetaTrader 5: press Ctrl+R to open Strategy Tester. Here's exactly how to configure it for meaningful results.
1
Choose your EA and Symbol
Expert Advisor: select your EA. Symbol: the pair your strategy is designed for (e.g. EURUSD). Period: must match what your EA uses (H1 if you built for H1). Model: select "Every Tick Based on Real Ticks" for the most accurate results. Avoid "Open Prices Only" for live trading strategies — it's faster but less accurate.
2
Set the date range to at least 2–3 years
Short backtests are almost meaningless. Use at least 2 years — ideally 5+ years — to capture multiple market regimes: trending markets, ranging markets, high volatility, low volatility, crises. If your strategy only looks good on 6 months of data, it's almost certainly curve-fitted.
3
Set realistic initial deposit and spread
Initial deposit: what you'd actually trade with ($1,000 or $10,000). Spread: set it to the real spread your broker charges (1.0–2.0 for EURUSD is realistic). A backtest with 0 spread is lying to you — it ignores one of your biggest costs.
4
Read the right metrics from results
Press Start. When it finishes, go to the Report tab. Know these key numbers — they tell the real story.
MetricWhat It MeansGood RangeRed Flag
Profit Factor Total Gross Profit ÷ Total Gross Loss. The single most important number. Above 1.5 Below 1.2 = barely profitable
Max Drawdown % Largest drop from peak equity to trough. How much pain you'd endure. Below 20% Above 30% = too risky for most
Win Rate Percentage of trades that were profitable. 40–70% depending on strategy type High win rate + low PF = large losses eating small wins
Sharpe Ratio Return per unit of risk. Higher = more consistent returns. Above 1.0 Below 0.5 = volatile returns
Total Trades Number of trades executed. Tells you if sample size is meaningful. Above 200 Below 50 = statistically unreliable
Recovery Factor Net Profit ÷ Max Drawdown. How fast it recovers from losses. Above 3.0 Below 1.0 = doesn't recover well
Ask AI to Analyze Your Results
My MQL5 EA backtest produced these results: - Symbol: EURUSD, H1 - Period: Jan 2020 – Jan 2025 (5 years) - Net Profit: $2,340 - Profit Factor: 1.38 - Max Drawdown: 24.5% - Total Trades: 187 - Win Rate: 52% - Sharpe Ratio: 0.71 Please analyze these results: 1. Is this a good strategy? Be honest. 2. What are the main weaknesses? 3. What specific changes to the code would likely improve these numbers? 4. What additional filters should I test?
⚠️
Overfitting Trap
If your EA shows 90%+ win rate and profit factor of 5.0+ on historical data but fails on live markets — you've overfitted. It "learned" the past data's noise instead of real patterns. Signs: too many input parameters, tested on too short a period, parameters that seem oddly specific (like RSI threshold of 67.3 instead of 70).
Chapter 05 — Fix It
Debugging smarter
with AI
Every EA will have bugs. The skill isn't avoiding them — it's fixing them fast. AI can diagnose and fix MQL5 bugs in seconds that would take hours manually.
Types of Bugs & How to Fix Each
Bug Type 01 — Compilation Error
SYMPTOMS: MetaEditor shows red text in Errors tab. EA won't compile. FIX PROMPT: "Fix this MQL5 compilation error. Code: [CODE] Error: [EXACT ERROR TEXT]" PRO TIP: Double-click the error in MetaEditor to jump to the exact line causing the problem.
Always copy the exact error text. "line 47: 'CTrade' - undeclared identifier" is far more useful than "it says there's an error."
Bug Type 02 — Logic Bug
SYMPTOMS: Compiles and runs but trades at wrong times, or never trades, or too many trades. FIX PROMPT: "My EA is [describe wrong behavior]. It should [describe expected behavior]. Here is the relevant logic section: [CODE BLOCK] What is wrong and how to fix it?" PRO TIP: Use Print() statements to log values and find where logic diverges from expectation.
Logic bugs are harder. Add Print() to log indicator values each tick, check them in MT5 Journal tab to spot where the logic fails.
Bug Type 03 — Runtime Error
SYMPTOMS: EA crashes during backtest or live run. MT5 Journal shows error codes like 4001, 4756. FIX PROMPT: "My MQL5 EA produced runtime error code [NUMBER] in the Journal. The error message is: [TEXT]. It happened during: [DESCRIBE WHEN]. Here is my code: [CODE]" PRO TIP: MQL5 error codes are documented at docs.mql5.com. Give the full code number to AI.
Runtime errors often come from invalid handles, arrays out of bounds, or trading on a closed market. AI can identify the exact cause quickly.
Bug Type 04 — Performance Bug
SYMPTOMS: EA works but backtest is very slow, or uses too much CPU. FIX PROMPT: "My MQL5 EA runs slowly. Please review this code for performance issues and rewrite any inefficient sections: [CODE] Focus on: CopyBuffer calls, loops, indicator recalculations." PRO TIP: The most common culprit is CopyBuffer inside loops or copying more data than needed.
Performance matters — a slow EA can miss trades in live conditions. Ask AI to specifically review your OnTick() function for unnecessary work.

Chapter 06 — Optimize It
Making your strategy
perform better
Optimization is the process of finding better parameter values and adding smarter logic. AI can guide every step — from running MT5's optimizer to interpreting results and suggesting code improvements.
MT5 Optimizer + AI
How to use the MT5 built-in optimizer
Strategy Tester has an "Optimization" mode that tests thousands of parameter combinations automatically. It finds which values of FastEMA, SlowEMA, RSI thresholds, etc. produce the best results. Here's how to use it intelligently with AI.
1
Ask AI what parameters to optimize and what range to test
Paste your code and ask: "Which input parameters should I optimize in MT5's Strategy Tester, and what range of values should I test for each? Keep the parameter space small to avoid overfitting." AI will give you a sensible range like EMA Fast: 5–20 step 1, EMA Slow: 15–50 step 5.
2
In Strategy Tester, enable "Optimize" and select "Slow complete algorithm"
Tick the checkbox on each input parameter you want to test. Set Start, Step, and Stop for each. Use "Slow Complete Algorithm" (tests every combination) for small parameter spaces. "Fast Genetic Based" for larger spaces — it's faster but less thorough. Optimize for "Profit Factor" not "Maximum Profit" — the latter is easily gamed.
3
Take the top results, validate on out-of-sample data
Critical step: the optimizer found best params for the optimization period. Now manually backtest those params on a DIFFERENT date range (out-of-sample). If performance collapses on new data, the parameters are overfit. If it holds, you may have found something real.
4
Ask AI to analyze the optimization results
Export the optimization results as CSV (right-click the results grid → Export). Paste the top 20 rows to AI and ask it to identify patterns, recommend robust parameters, and explain what the data shows about your strategy's behavior.
Code-Level Optimizations
AI-powered code improvements
Beyond parameter tuning, AI can help you improve the actual strategy logic. These are the prompts that unlock real performance gains.
Add Time Filter
"Add a trading hours filter to my EA. Only trade between 08:00–12:00 and 13:00–17:00 UTC (London+NY session overlap). No trades outside these hours even if a signal fires. Use MqlDateTime for time parsing."
Many strategies work only in specific sessions. Session overlap has the most volume and tightest spreads.
Add Trend Filter
"Add an ADX filter to my EA. Only open new positions when ADX(14) is above 25, confirming a trending market. Skip trades when ADX is below 25 (ranging). Add ADX period as input param."
Trading trend strategies in ranging markets is the #1 source of losses. ADX filters this ruthlessly.
Add Trailing Stop
"Replace the fixed take profit with a trailing stop. The stop should trail 1.5x ATR(14) behind the highest price reached since entry. Update the stop in OnTick() only when price moves in our favor."
Trailing stops let winners run while protecting profits. Often better than fixed TP for trend-following.
Add Break-Even
"Add break-even logic: when price moves 1x ATR in profit, move the stop loss to entry price plus 2 pips (to cover spread). This ensures the trade cannot become a loss once it moves far enough in our favor."
Break-even logic turns potential losses into scratch trades. Dramatically improves the psychological experience of live trading.
Add Higher TF Filter
"Add a higher timeframe trend filter. Before entering a buy on H1, check that the EMA(50) on H4 is rising (current value above value 3 bars ago). Only buy when higher TF trend aligns with signal. Add higher TF period as input."
Multi-timeframe confluence is one of the most powerful filters. Only trade with the higher timeframe trend behind you.
Add Volatility Guard
"Add a volatility filter: skip trading if the current ATR(14) is more than 2x the 50-period average ATR (abnormally high volatility). Also skip if ATR is below 0.5x the average (abnormally low). Only trade in 'normal' volatility."
News spikes and dead markets are where most EAs break. This filter automatically avoids both extremes.

Risk management
code patterns
The best strategy in the world will blow your account without proper risk management code. These are the prompts and patterns you must implement before going live.
Essential Risk Code

// ── PATTERN 1: Dynamic Lot Sizing (Risk % per trade) ── // Risk exactly X% of account on each trade — not a fixed lot size // Ask AI: "Add a RiskPercent input and calculate lot size from it" double CalcLotSize(double riskPct, double slPips) { double balance = AccountInfoDouble(ACCOUNT_BALANCE); double riskAmount = balance * riskPct / 100.0; double pipValue = SymbolInfoDouble(_Symbol, SYMBOL_TRADE_TICK_VALUE); double lotSize = riskAmount / (slPips * pipValue); lotSize = MathMax(0.01, MathMin(lotSize, 10.0)); // clamp return(NormalizeDouble(lotSize, 2)); } // ── PATTERN 2: Daily Loss Limit ──────────────────────── // Stop trading for the day if account drops X% since midnight // Ask AI: "Add a DailyLossLimit input — stop trading if daily // loss exceeds this percentage" double startOfDayBalance; bool IsDailyLossExceeded(double limitPct) { double currentEquity = AccountInfoDouble(ACCOUNT_EQUITY); double dailyLoss = (startOfDayBalance - currentEquity) / startOfDayBalance * 100.0; return(dailyLoss >= limitPct); } // ── PATTERN 3: Max Drawdown Kill Switch ──────────────── // Disable EA completely if drawdown from account peak exceeds X% // Ask AI: "Add a MaxDrawdown input — disable trading permanently // if drawdown from peak balance exceeds this %" double peakBalance; bool IsDrawdownExceeded(double maxPct) { double balance = AccountInfoDouble(ACCOUNT_BALANCE); if(balance > peakBalance) peakBalance = balance; double dd = (peakBalance - balance) / peakBalance * 100.0; return(dd >= maxPct); } // ── PATTERN 4: Max Open Trades Limit ────────────────── // Never open more than N positions at once if(PositionsTotal() >= MaxOpenTrades) { return; // Skip this tick — already at max positions } // ── PATTERN 5: Check margin before trading ───────────── // Don't open trades if margin level is too low double marginLevel = AccountInfoDouble(ACCOUNT_MARGIN_LEVEL); if(marginLevel < 200.0 && marginLevel > 0) // below 200% { Print("WARNING: Margin level too low: ", marginLevel, "%"); return; }
🛑
The 5 Risk Rules — Never Skip These
1. Never risk more than 1–2% of account per trade — use dynamic lot sizing
2. Always have a hard stop loss — never trade without one
3. Set a daily loss limit — if you lose 3–5% in a day, stop trading
4. Set a maximum drawdown kill switch — if you lose 15–20% from peak, disable the EA
5. Never trade on a live account what you haven't tested on demo for 30+ days
Chapter 08 — Level Up
Advanced AI workflows
for serious traders
Once you've mastered the basics, these advanced prompting and workflow techniques will take your EA quality from "it works" to "it's genuinely good."
Multi-Step Strategy Development
The "Build in Layers" approach
Don't build the full strategy in one prompt. Build it in layers — each layer adds one feature. This keeps code clean and lets you pinpoint exactly what change helped or hurt performance.
1
Layer 1: Bare bones entry/exit only
Just the signal. No filters, no risk management beyond a basic SL. Get this compiling and backtesting. Establish your baseline Profit Factor and Drawdown. This is your control group.
2
Layer 2: Add risk management
Add dynamic lot sizing, proper ATR stop loss, break-even. Backtest again. Compare to baseline. If drawdown dropped significantly, the risk code is working. If profit factor dropped, the stops might be too tight.
3
Layer 3: Add the best performing filter
Add ONE filter (e.g. time filter). Backtest. Compare. Did it help? If yes, keep it. If performance got worse, remove it and try a different filter. Never add multiple filters at once — you won't know which one helped.
4
Layer 4: Walk-forward validation
Divide your data into thirds. Optimize on the first two thirds. Test on the final third without touching any parameters. This is the closest thing to a "real future" test you can do on historical data.

Chapter 09 — Ship It
Pre-live deployment
checklist
Before you switch from demo to live — go through every item below. Click each one as you complete it. Don't skip a single item. This checklist has been built from the mistakes traders make most often.
Code Quality
EA compiles with 0 errors and 0 warnings in MetaEditor. Warnings are not harmless — they indicate potential problems.
All indicator handles are checked for INVALID_HANDLE in OnInit(), and the EA returns INIT_FAILED if any handle fails.
OnDeinit() releases all indicator handles using IndicatorRelease(). Memory leaks crash MT5 after hours of running.
CopyBuffer return value is checked every time — if it returns less data than requested, the tick is skipped cleanly.
You can explain every single line of code. Ask AI to explain anything unclear. If you can't explain it, you can't trust it.
Strategy Validation
Backtested on at least 3 years of data with realistic spread values (not 0). More data = more reliable results.
Profit Factor above 1.4 across multiple time periods, not just the best one. Consistent PF beats one amazing period.
Maximum drawdown under 25% in backtest. Live trading always has higher drawdown than backtest due to slippage.
Strategy performs on out-of-sample data — tested on a date range NOT used for optimization. Performance degradation under 30% is acceptable.
More than 200 total trades in the backtest. Fewer trades = results could be luck. Statistical significance requires sample size.
Risk Management
Every trade has a stop loss. Non-negotiable. Even a wide one. An EA without a stop loss is a time bomb.
Lot size is set to risk 1–2% of account per trade, not a large fixed number. On a $1,000 account, max risk per trade = $20.
Daily loss limit is implemented and tested. If you lose 3% in a day, the EA stops trading until midnight.
Maximum drawdown kill switch is set. If account drops 15% from peak, EA disables itself and you review manually.
Live Deployment
Ran on demo account for at least 30 days. Demo results must roughly match backtest behavior — if not, investigate why before going live.
VPS (Virtual Private Server) is set up if running 24/7. Your home computer sleeping or losing internet = missed trades and open positions with no management.
Notifications are set up — MT5 can send push notifications or emails when trades open/close. You should know what your EA is doing.
You have a monthly review process. Check live performance vs backtest monthly. If live PF falls below 1.1 for 3 consecutive months, pause and reassess.
