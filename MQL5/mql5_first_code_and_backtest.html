The Complete EA —
Every Line Explained
Copy this full code into a new EA file in MetaEditor (File → New → Expert Advisor). Then read the breakdown below to understand exactly what each part does.
MQL5 — MACrossoverEA.mq5
#property copyright "MQL5 Tutorial" #property version "1.00" #property strict // ─── INPUTS ────────────────────────────────────────── input int FastPeriod = 9; // Fast MA period input int SlowPeriod = 21; // Slow MA period input double LotSize = 0.1; // Trade lot size // ─── GLOBAL VARIABLES ──────────────────────────────── int fastHandle, slowHandle; double fastBuffer[], slowBuffer[]; // ─── ONINIT ────────────────────────────────────────── int OnInit() { fastHandle = iMA(_Symbol, _Period, FastPeriod, 0, MODE_EMA, PRICE_CLOSE); slowHandle = iMA(_Symbol, _Period, SlowPeriod, 0, MODE_EMA, PRICE_CLOSE); if(fastHandle == INVALID_HANDLE || slowHandle == INVALID_HANDLE) { Print("❌ Failed to create MA handles!"); return(INIT_FAILED); } ArraySetAsSeries(fastBuffer, true); ArraySetAsSeries(slowBuffer, true); Print("✅ EA started. Fast=", FastPeriod, " Slow=", SlowPeriod); return(INIT_SUCCEEDED); } // ─── ONTICK ────────────────────────────────────────── void OnTick() { // Copy indicator data into arrays (3 bars back) if(CopyBuffer(fastHandle, 0, 0, 3, fastBuffer) < 3) return; if(CopyBuffer(slowHandle, 0, 0, 3, slowBuffer) < 3) return; // Current bar [0] and previous bar [1] double fastNow = fastBuffer[0]; double fastPrev = fastBuffer[1]; double slowNow = slowBuffer[0]; double slowPrev = slowBuffer[1]; // Check for crossover signals bool crossUp = (fastPrev <= slowPrev) && (fastNow > slowNow); bool crossDown = (fastPrev >= slowPrev) && (fastNow < slowNow); if(crossUp) { CloseAllTrades(); OpenBuy(); } else if(crossDown) { CloseAllTrades(); OpenSell(); } } // ─── HELPER FUNCTIONS ──────────────────────────────── void OpenBuy() { CTrade trade; double ask = SymbolInfoDouble(_Symbol, SYMBOL_ASK); if(trade.Buy(LotSize, _Symbol, ask, 0, 0, "MA Cross Buy")) Print("🟢 Buy opened at ", ask); else Print("❌ Buy failed: ", trade.ResultRetcodeDescription()); } void OpenSell() { CTrade trade; double bid = SymbolInfoDouble(_Symbol, SYMBOL_BID); if(trade.Sell(LotSize, _Symbol, bid, 0, 0, "MA Cross Sell")) Print("🔴 Sell opened at ", bid); else Print("❌ Sell failed: ", trade.ResultRetcodeDescription()); } void CloseAllTrades() { CTrade trade; for(int i = PositionsTotal() - 1; i >= 0; i--) { ulong ticket = PositionGetTicket(i); if(PositionGetString(POSITION_SYMBOL) == _Symbol) trade.PositionClose(ticket); } } void OnDeinit(const int reason) { IndicatorRelease(fastHandle); IndicatorRelease(slowHandle); Print("👋 EA stopped."); }

Section 1 — Inputs & Global Variables
Lines 1–12
①
#property strict
Turns on strict compiler mode. Forces MQL5 to catch more bugs at compile time — like undeclared variables or type mismatches. Always include this line.
Always include
②
input int FastPeriod = 9
The input keyword makes this variable user-editable in MT5's EA settings dialog. The default is 9, meaning the EMA averages the last 9 candles. Users can change it without touching code.
Makes EA flexible
③
int fastHandle, slowHandle
Handles are like ID numbers for indicator objects. When you call iMA() it creates an indicator internally and gives you a handle to reference it later when copying data.
How MT5 indicators work
④
double fastBuffer[], slowBuffer[]
Arrays to store indicator values. fastBuffer[0] = current bar's fast MA value. fastBuffer[1] = previous bar's value. We use [0] and [1] to detect crossovers.
Key pattern in all MQL5 indicators
Section 2 — OnInit()
Runs once at startup
①
iMA(_Symbol, _Period, FastPeriod, ...)
iMA() creates an EMA indicator. _Symbol = current chart's pair (e.g., EURUSD). _Period = current chart's timeframe. FastPeriod = the input value. Returns a handle number.
Creates the indicator object
②
if(fastHandle == INVALID_HANDLE)
Error handling. If the indicator failed to create (e.g., not enough history data), we print an error and return INIT_FAILED. This stops the EA from running with broken data.
Always check handles
③
ArraySetAsSeries(fastBuffer, true)
This reverses the array index direction so that [0] = most recent bar (current) and [1] = previous bar. Without this, the array is in the opposite order and your crossover logic would be backward.
Critical — easy to forget
Section 3 — OnTick() Crossover Detection
The heart of the EA
①
CopyBuffer(fastHandle, 0, 0, 3, fastBuffer)
Copies 3 data points from the indicator into our array. The return value is how many values were actually copied. If we get fewer than 3, there's not enough data yet — we return early to wait.
Gets indicator values each tick
②
bool crossUp = (fastPrev <= slowPrev) && (fastNow > slowNow)
Crossover detection logic. A bullish crossover happened if: on the PREVIOUS bar, fast was at or below slow — AND on the CURRENT bar, fast is now above slow. Both conditions must be true.
The core trading signal

Common errors & what they mean
Decode the most frequent beginner errors
error: 'xxx' - undeclared identifier
You used a variable or function that doesn't exist or wasn't included. Check spelling. For CTrade, add #include <Trade/Trade.mqh> at the top.
error: semicolon expected
A line is missing a semicolon at the end. Double-click the error in the Toolbox to jump to the exact line.
INVALID_HANDLE in Experts tab
iMA() failed to create the indicator. Usually means not enough historical data on that symbol/timeframe, or wrong parameters passed.
Sad face ☹ on chart (no smiley)
Algo trading is disabled. Click the "Algo Trading" button in the MT5 toolbar (make it turn green), then re-attach the EA.
Part 2 · Backtesting — Section 1
Open the
Strategy Tester
The Strategy Tester lets you run your EA against years of historical price data in seconds. It simulates every price tick as if you were trading live — but in the past.
A
What is backtesting?
Before you open the tester
→
Backtesting replays historical market data through your EA as if it were running live. Your EA's OnTick() function is called for every historical tick.
→
The result shows you simulated profit/loss, win rate, drawdown, and a trade-by-trade history — all from the past without risking real money.
⚠️
Past performance ≠ future results. A strategy that worked historically may fail live. Backtests have limitations: they don't account for slippage, broker spread changes, or market regime shifts. Use them to validate logic, not to predict profits.
B
Open the Strategy Tester
3 ways to access it
1
Keyboard shortcut (fastest): Press Ctrl+R in MT5.
2
Menu: Go to View → Strategy Tester
View›Strategy Tester
3
Toolbar: Click the Strategy Tester icon (looks like a bar chart with a play button).
4
The Strategy Tester panel opens at the bottom of MT5. It has several tabs — you'll mainly use Settings and Results.

Part 2 · Backtesting — Section 3
Run Your First
Backtest
Hit Start and watch MT5 simulate a full year of trading in seconds.
A
Start the backtest
1
With your settings configured, click the Start button (green play icon) in the Strategy Tester.
2
A progress bar appears. For 1 year of EURUSD H1 data, it typically finishes in 5–30 seconds depending on your computer.
3
While it runs, you'll see a simulated chart animating the price replay. You can click Visual Mode to watch trades open and close in real time — useful for spotting issues.
💡
First run taking too long? MT5 needs to download historical data the first time. If the bar is stuck, check the Journal tab — you may see "downloading history" messages. Wait for it to finish.
B
What happens during the test
Behind the scenes
→
MT5 loads historical EURUSD H1 tick data for 2023. It then calls your OnTick() function for every single price movement — exactly as if you were live trading.
→
When your EA calls CTrade.Buy() or CTrade.Sell(), MT5 simulates the order opening at the historical price at that moment — including spread.
→
Every simulated trade is recorded with entry price, exit price, profit/loss, and time. All this data feeds the Results tab you'll see next.

X.XX
Risk-adjusted return. >1.0 is decent. >2.0 is excellent.

How to interpret your first result
Expected outcome for a simple MA crossover EA
→
Probably a small loss or break-even. MA crossovers without stop losses tend to underperform on their own. This is normal — the point was to learn the code and backtest process, not to find a winning strategy.
→
Look at the trade list. Do you see trades opening and closing as expected? Are buy trades followed by sell trades? If yes — your EA logic is working correctly.
→
Check the Experts tab inside the tester (bottom of Strategy Tester) — your Print() messages from the backtest appear there too.
💡
Want to improve the results? Try adding a Stop Loss and Take Profit to each trade. In the OpenBuy() function, replace the 0, 0 parameters with actual SL/TP values (in price units, not pips).

You've built and
backtested your first EA 🎉
Here's what to tackle next to keep levelling up your MQL5 skills.
Improve this EA first
1
Add Stop Loss & Take Profit — Replace the 0, 0 in OpenBuy/OpenSell with real SL/TP values. For EURUSD, 50 pips SL = price ± 0.00500.
2
Add a spread filter — Only open trades if the current spread is below a threshold. High spreads during news events can destroy MA crossover strategies.
3
Trade only on closed bars — Check if we're on a new bar before evaluating signals. This prevents multiple signals on the same candle. Use static datetime lastBar and compare iTime().
4
Add a maximum trades limit — Count open positions with PositionsTotal() before opening a new one.
What to learn next
📊
More Indicators
Learn iRSI(), iBands(), iMACD(). The pattern is the same: get a handle, copy buffer, read values.
🛡️
Risk Management
Learn to size positions as a % of account balance instead of a fixed lot size. This is essential for real trading.
🔧
Optimization
Use Strategy Tester's Optimize mode to find the best input parameters. Watch out for overfitting.

Walk-Forward Testing
Test on data the EA wasn't optimized on. The real validation of any strategy before risking money.
