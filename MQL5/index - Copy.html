Chapter 02 — Process
The 5-Step Loop
every pro uses
Vibe coding isn't random prompting. It follows a repeatable loop: Describe → Generate → Test → Iterate → Deploy. Master this loop and you can build anything.
01
Describe your strategy in plain English
Before touching AI, write out what your EA should do in 3–5 sentences. Entry condition, exit condition, risk management, which timeframe. The clearer your description, the better the code.
YOU DO THIS
02
Generate the initial code with AI
Use a structured prompt (templates in Chapter 4) to get AI to write the full EA. Request comments on every section, error handling, and input parameters.
AI DOES THIS
03
Compile, attach, verify logic
Compile in MetaEditor (F7). Fix any errors by pasting them back to AI. Attach to a demo chart and check the Experts tab — does the EA behave as described? Check entries and exits manually on a few candles.
YOU DO THIS
04
Backtest and send results to AI
Run a backtest in Strategy Tester. Copy the key metrics (Net Profit, Drawdown, Win Rate, Profit Factor) and paste them back to AI. Ask it to suggest improvements.
BOTH
05
Iterate until satisfied, then forward-test
Repeat the loop. Once backtests look reasonable, run the EA on a demo account for 2–4 weeks to see real-time behavior before touching a live account.
BOTH

Chapter 08 — Optimization
Optimize smarter,
not just harder
MT5's optimizer can test thousands of parameter combinations. AI helps you choose which parameters to optimize, interpret the heatmap, and avoid the deadly trap of overfitting.
// how MT5 optimization works
⚙️
Setting up an optimization run
Strategy Tester → Settings → Optimization
1
In Strategy Tester Settings, change the mode from Single Pass to Slow Complete Algorithm or Fast Genetic Algorithm (faster, good for 3+ parameters).
2
In the Inputs tab, check the box next to each parameter you want to optimize. Set Start, Step, and Stop values. Example: FastPeriod Start=5, Step=1, Stop=20.
3
Set the Optimization criterion — use Balance + max Sharpe Ratio for a balanced result. Avoid optimizing for pure profit, it ignores risk.
4
Click Start. MT5 runs all combinations and sorts results by your criterion. The top row = best parameters found.
⚠️
Never optimize more than 3 parameters at once. With 10 parameters each having 10 values = 10 billion combinations. Stick to 2–3 at a time. Optimize what matters most first.
// overfitting — the silent killer
💀
Overfitting is when your EA is perfectly tuned to historical data but fails in live trading. The optimizer found parameters that "fit" the past noise, not the underlying pattern. The result: incredible backtests, terrible live performance.
📋 Optimization Analysis Prompt
I ran an MT5 optimization for my EA. Here are the top 10 results (parameter combinations and their profit/drawdown):

[paste the top 10 rows from the MT5 optimization results table]

My strategy: [brief description]

Please:
1. Which parameter combination would you recommend — the highest profit or one of the middle ones? Why?
2. Are the best results clustered (good sign) or all over the place (bad sign = overfitting)?
3. What does it mean if the #1 result is very far from the #2 result in parameter values?
4. How should I validate these parameters to make sure I'm not overfitting?
// walk-forward testing — the real validation
IS
In-Sample (Optimization Period)
2021–2022 — Optimize parameters on this data. Find the best FastPeriod=11, SlowPeriod=23.
OPTIMIZE HERE
OOS
Out-of-Sample (Validation Period)
2023 — Run a single backtest with those exact parameters on data the optimizer never saw. If it still performs well → the strategy is real. If it fails → it was overfitted to 2021–2022 noise.
VALIDATE HERE
✅
The rule: Only trust a strategy if the out-of-sample (OOS) result is at least 60% as good as the in-sample result. If OOS is much worse, the optimization found noise, not signal.

Chapter 09 — Performance
Make your EA
blazing fast
OnTick() can run hundreds of times per second. Slow code = missed signals, slippage, and broker disconnects. These AI-assisted techniques will make your EA production-grade fast.
// the #1 performance fix: new bar detection
⚡
Most beginner EAs run all their logic on EVERY tick. On a busy market, that's hundreds of times per second. 99% of that is wasted — your crossover condition doesn't change 200 times per second. Run heavy logic only once per new bar.
MQL5 — New Bar Guard (add to every EA)
// ─── In global scope ─────────────────────────── static datetime lastBarTime = 0; // ─── At the TOP of OnTick() ───────────────────── void OnTick() { // Only run logic once per new bar (not every tick) datetime currentBar = iTime(_Symbol, _Period, 0); if(currentBar == lastBarTime) return; lastBarTime = currentBar; // ← All your heavy logic goes here // This now runs once per bar, not hundreds of times }
// more performance wins
🔒
Cache indicator handles
Create handles once in OnInit(), not inside OnTick(). Creating an indicator handle every tick is extremely expensive and will eventually return INVALID_HANDLE.
📦
Copy only what you need
Use CopyBuffer(handle, 0, 0, 3, arr) to copy only 3 values. Never copy 1000 bars of history when you only need the last 2.
🧹
Release handles on exit
Always call IndicatorRelease(handle) in OnDeinit(). MT5 has a limit on open indicators per chart — leaking them causes crashes.
🛑
Early return guards
Put your cheapest checks first. If spread is too high, return immediately before doing any indicator calculations. Gate expensive work behind cheap gates.
